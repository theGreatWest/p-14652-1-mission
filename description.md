스프링 : 내부 이벤트
카프카 : 외부 이벤트

카프카
이벤트를 외부로 보내고, 리스너로 카프카에서 받아오는 구조
배포할 때, 도커 컨테이너로 따로 띄움

테스트환경
임베디드로 처리

### 용어 정리
- 토픽 : 레코드들이 저장되는 별도의 저장소(MySQL의 테이블과 유사)
- 레코드 : 카프카에서는 이벤트를 레코드라고 부른다.
- 프로듀서 : 이벤트를 생성하는 주체(eventPublisher.publishEvent(...))
- 컨슈머 : 이벤트를 수신하는 주체(@KafkaListener(topics = "MemberJoinedEvent"))

$ docker ps
명령어로 현재 실행 중인 컨테이너 확인 가능

도커 데스크톱 작동 필수
compose.yaml 파일을 통해 카프카 컨테이너와 레드판다 콘솔 컨테이너 한 번에 실행
서비스탭에서 작동 확인
- STOP : 컨테이너 중지(볼륨의 데이터(토픽/레코드 등)와 나머지 데이터도 유지된다)
- DOWN : 컨테이너 삭제(볼륨의 데이터(토틱/레코드 등)는 유지된다)

### 레드판다 콘솔 접속
레드판다 콘솔 - http://localhost:8090
